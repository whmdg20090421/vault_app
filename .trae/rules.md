# 天眼·艨艟战舰 发版规则 (Release Rules)

1. **版本更新同步**：每次发布新版本（例如修改 `pubspec.yaml` 中的 `version` 字段或修改 `CHANGELOG.md`）时，**必须**同步检查并确保应用内“关于”页面的版本号显示正确。
2. **更新记录自动同步**：应用内的“更新日志”（Changelog）页面必须与项目根目录的 `CHANGELOG.md` 保持内容一致（目前已实现读取 Assets 自动同步）。发版时必须将新的特性、修复项记录到 `CHANGELOG.md` 的顶部。
3. **安全与隐私目录规范**：在处理内部私有目录（`data/data/...`）和外部私有目录（`0/android/data/...`）时，同一类的数据文件必须归类放置在相应的文件夹中。除非引发乱码或Bug，否则必须使用**清晰的中文命名**（如 `应用配置/`、`主题背景/`、`加密任务记录/`、`运行日志/`），以便用户可以通过文件管理器直观地找到并管理自己的数据。
4. **自动化打包与发布检测规范**：当被要求发布 APK 或 Tag 时，必须持续监测远端构建流水线，并确认最终产品真实可用后才能结束任务。必须遵循以下流程：
   - 第一步：将本地代码推送到远端仓库。
   - 第二步：**必须实时使用浏览器监测它的编译状态**。如果编译报错，必须使用浏览器或其它工具就地定位日志、修改报错并重新推送。
   - 第三步：只有当编译成功之后，**必须使用浏览器前往安装包发布页（Releases/Tags）去检查是否发布，并截取快照之后才能结束**。若流水线编译失败或没有生成安装包，则严禁结束运行。
   - *注：任何凭证（如账号、密码、Token）仅保存在本地，严禁上传到 GitHub。*
