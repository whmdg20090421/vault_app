# 加密文件处理模块 V4 重构 Spec

## Why
原有的加密逻辑已被清理，现需要基于最新完整版 V4 需求重构核心加密流与进度展示。新架构要求极高的进度透明度、细粒度的层级树状任务控制（参考 Cryptomator），以及高性能的并发加密和动态 UI 刷新机制，以解决历史版本中任务状态混乱、进度计算不准确等痛点。

## What Changes
- **一、文件导入与路径获取**：
  - 用户点击「导入加密文件夹」或「导入加密文件」按钮，唤起系统原生文件选择界面。
  - 选择确认后，提交绝对路径至核心模块，创建独立加密任务。
- **二、加密文件结构树定义与构建**：
  - 每个任务对应独立根节点，统一存储全局任务列表，JSON持久化至内部私有数据目录，递归展开为文件树。
  - 文件夹节点：仅维护层级结构。含 `taskId`（仅根节点）、`name`、`type` (固定 folder)、`isPaused`、`children`。
  - 文件节点：含 `name` (完整文件名)、`type` (固定 file)、`size` (自动换算1~1000间，保留1位小数的 B/KB/MB/GB/TB)、`rawSize` (原始字节)、`status` (五种状态)、`isPaused` (继承父节点)。
  - 五种状态枚举：`pending_waiting` (未加密·等待中), `pending_paused` (未加密·已暂停), `encrypting` (正在加密), `completed` (已完成), `error` (错误·等待修复)。
  - 构建时生成唯一 `taskId`，同步读取 `rawSize` 换算 `size`，初始 `isPaused=false`，文件初始 `pending_waiting`，完成后写入本地并触发全局刷新。
- **三、多线程加密执行**：
  - 读取预设加密线程数，并行处理 `status=pending_waiting` 且 `isPaused=false` 的文件。
  - 中断处理：应用退出/强制终止导致中断，重置为 `pending_waiting`。重启后自动读取并重新完整加密，不支持单文件断点续传。
  - 异常处理：检测缺失/损坏/无法读取时，终止该文件进程，标为 `error`，抛出异常UI提示手动修复，不影响其他文件。
- **四、加密状态更新与进度恢复**：
  - 开始加密即更新 `encrypting`，完成后更新 `completed`。
  - 重启时自动读取全局列表恢复进度，继续未暂停的未完成任务。自动扫描原文件，不存在的更新为 `error`。
- **五、主页导航栏加密信息总览**：
  - 导航栏展示小型四色进度条图标，点击显示详细数值。
  - 四色：绿(已加密), 黄(正在加密), 红(未加密等待中), 灰(未加密已暂停+错误)。四者合计100%，基于各状态 `rawSize` 之和计算。数值提示：已加密大小 / 总大小。
  - **强制可视化规则**：占比小于1%强制保持1像素最小显示长度，不影响其他颜色比例。
- **六、层级化加密进度详情查看**：
  - 主界面「查看同步进度」按钮，下往上滑出半屏模态框。展示根节点同级关系，支持多级嵌套展开。
  - 条目三行布局：1. 暂停/继续圆形按钮 + 名称 (异常带红感叹号)；2. 极细四色进度条 (含子目录总字节占比)；3. 左侧整体加密百分比 (保留1位小数) / 异常显示「存在异常」，右侧统计 (文件 1/1，文件夹 已加密/总数)。
  - 暂停/继续功能：点击暂停终止该条目及子节点所有加密线程，`isPaused=true`，正在加密的转为 `pending_paused`。继续则转回 `pending_waiting`。父节点覆盖子节点，子节点不影响父/同级。
  - 长按删除功能：长按弹菜单「移除加密」(异常带「标记为已修复」)。仅删除被长按的节点及子节点，不影响同级/父节点。立即终止线程，从列表永久移除并更新进度。二次确认。修复功能则重置 `error` 为 `pending_waiting`。
- **七、动态进度刷新机制（核心优化）**：
  - 前台状态（显示进度界面）：全量刷新后，切为 0.5~1秒/次的高频刷新。
  - 后台静默（关闭进度界面）：切为 5~10秒/次的低频刷新。不重绘UI，仅更新内存并持久化。
  - 无活跃任务（全 `completed`）：执行最终全量刷新，缓存至本地，停止所有自动刷新任务。打开界面直接读缓存。新任务创建才重启。
  - 独立监听线程负责计算更新，不直接操作UI，自适应频率，确保持久化。
- **八、任务完成与自动归档**：
  - 根下所有文件 `completed` 即判定完成。执行目录存在性检查。
  - 均存在则移至加密历史记录永久保存；缺失则保留活跃列表，缺失文件标 `error`，提示手动处理。
  - 历史记录与进行中任务布局一致（隐藏进度/控制按钮，显示时间）。长按支持二次确认「删除历史」。仅删进度，不删文件和索引。
- **九、加密完成后处理**：
  - 文件加密完成，计算哈希，将 哈希与 (taskId+相对路径) 关联写入本地哈希索引库 (`local_index.json`)，用于完整性校验和解密。

## Impact
- Affected specs: 加密任务调度核心、加密进度 UI 展示、文件导入处理流、主页导航栏。
- Affected code:
  - `lib/encryption/models/encryption_node.dart` (新增)
  - `lib/encryption/services/encryption_task_manager.dart` (重写)
  - `lib/encryption/vault_explorer_page.dart` (重写导入逻辑)
  - `lib/home/home_page.dart` (修改导航栏)
  - `lib/encryption/widgets/encryption_progress_modal.dart` (新增)

## ADDED Requirements
### Requirement: 细粒度层级状态树与完全还原V4规格
系统必须严格遵循V4需求的所有细节，从导入构建、四色进度计算、最小1%展示、自适应动态刷新频率、局部暂停覆盖、长按剥离、归档检测到后处理哈希写入，无一遗漏。
