- [ ] 导入文件夹或文件后，系统能够正确解析为 `EncryptionNode` 的树结构（包含唯一的 `taskId` 和 `type=folder/file` 等）。
- [ ] 能够根据并发设置拉取 `pending_waiting` 状态的文件进行多线程加密，完成后正确设为 `completed` 并持久化哈希至 `local_index.json`。

- [ ] 前台有进度 UI 活跃时，全局进度每 0.5~1 秒刷新一次；关闭所有相关 UI 后，后台以 5~10 秒频率刷新；全任务完成则停止刷新并缓存进度。
- [ ] 首页顶部导航栏存在四色进度条图标，展示各状态总 `rawSize` 占比，任意部分 >0 时必须保持 1%（1像素）的显示宽度。
- [ ] 主界面能够滑出半屏“查看同步进度”模态框，列出所有任务根节点，并支持树状层级展开。
- [ ] 进度条目采用三行紧凑布局：名称+状态标识、四色细进度条、百分比及完成数量统计。
- [ ] 用户可以点击条目的暂停/继续按钮，成功停止或恢复该节点及其子树的加密线程；长按支持彻底移除该节点或标记错误为已修复。
