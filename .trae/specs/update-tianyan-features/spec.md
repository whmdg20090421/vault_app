# Tianyan Features Update Spec

## Why
为提升用户体验，需在应用中补充版本信息及更新日志（手风琴交互）、直观的加密数据统计饼图、以及详尽的树状加密任务进度面板。此外，需明确并优化工作流触发规则，以节约 GitHub Actions 的 CI 资源。

## What Changes
- **工作流规则**：重申并遵守在开发过程中不主动执行 git commit / push 的限制，避免耗尽 CI 资源。
- **关于与更新日志**：在设置页顶部新增"关于"入口；"关于"页展示应用名、版本号等基本信息，并提供"更新日志"入口；"更新日志"页采用手风琴（Accordion）交互列出历史版本详情。
- **首页概览饼图**：在主页新增半宽数据卡片，内置饼图展示已加密与未加密（含失败、暂停、尚未加密）的文件大小与百分比，数值自动换算 MB/GB。
- **加密任务进度面板**：加密页右上角新增动态入口；面板支持根目录列表层级视图（点击单根目录进入专注视图）；文件夹支持目录树展开，包含极细底部进度条及文件数/容量百分比统计。

## Impact
- Affected specs: 设置模块视图、主页概览视图、加密任务状态与进度展示视图。
- Affected code: 
  - 设置页相关 UI 组件（如 `settings_page.dart`）。
  - 主页相关 UI 组件（如 `home_page.dart`）。
  - 加密页与进度面板组件（如 `encryption_page.dart` 及新的进度面板 Widget）。
  - 依赖配置 `pubspec.yaml`（引入饼图所需库，如 `fl_chart`）。

## ADDED Requirements
### Requirement: 提交行为限制
在日常功能实现过程中，系统（智能助手）SHALL NOT 自动执行 `git commit` 或 `git push`，必须仅在用户发出"编译APK"或"调试APK"等明确指令时执行。

### Requirement: 关于页面与手风琴日志
系统 SHALL 在设置页提供"关于"页面展示基础属性，并包含"更新日志"子页面。更新日志 SHALL 支持手风琴模式折叠与展开各个版本的详情。

### Requirement: 主页数据概览模块
系统 SHALL 在主页提供一个宽度占 1/2 的卡片，通过饼图可视化当前存储状态。饼图 SHALL 明确区分已加密与未加密区域，并包含动态计算的 MB/GB 单位文本与百分比。

### Requirement: 加密进度多级面板
系统 SHALL 仅在加密任务活动时，在加密页右上角展示入口。面板 SHALL 能够隔离显示不同的根目录任务，支持目录树层层下钻。每一级文件夹项 SHALL 底部附带铺满宽度的极细进度条，以及左侧的文件数进度和右侧的容量百分比。
