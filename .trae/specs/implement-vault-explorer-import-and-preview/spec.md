# 保险箱文件导入与文件预览 Spec

## Why
当前“保险箱文件浏览器”中仍存在标记为“开发中”的能力（导入明文文件/文件夹、读取文件）。这会阻断用户对保险箱的基本使用闭环，因此需要将这些待开发功能落地为可用实现。

## What Changes
- 实现“导入明文文件”：将用户选择的本地明文文件导入到当前保险箱路径，并按当前保险箱配置自动加密（若启用文件名加密/内容加密）。
- 实现“导入明文文件夹”：递归导入整个文件夹到当前保险箱路径，保留目录结构。
- 实现“文件读取/预览”：点击文件后提供预览/导出等操作，不再提示“开发中”。
- 引入基础的导入/导出进度反馈与取消能力，避免长时间操作造成用户误判卡死。

## Impact
- Affected specs: Encryption Vault Management
- Affected code:
  - `lib/encryption/vault_explorer_page.dart`（导入、预览、导出 UI 与交互）
  - `lib/vfs/virtual_file_system.dart`、`lib/vfs/encrypted_vfs.dart`、`lib/vfs/local_vfs.dart`（复用现有 VFS 能力进行上传/读取）
  - 可能新增：`lib/encryption/services/`（将导入/导出逻辑从 UI 层抽离的轻量 service）

## ADDED Requirements
### Requirement: 导入明文文件
系统 SHALL 支持在保险箱浏览器中将本地明文文件导入到当前目录。

#### Scenario: 成功导入多个文件
- **WHEN** 用户在保险箱浏览器点击“导入明文文件”并选择一个或多个文件
- **THEN** 系统将文件依次写入到当前保险箱路径下（保持原文件名）
- **AND** 若保险箱启用加密域（例如启用文件名加密/内容加密），则导入过程必须通过 VFS 进行透明加密写入
- **AND** 导入完成后刷新当前目录列表

#### Scenario: 名称冲突
- **WHEN** 目标路径已存在同名文件
- **THEN** 系统按“最小实现”策略处理：默认覆盖（覆盖前删除旧文件或直接写入覆盖，具体由底层实现决定）

### Requirement: 导入明文文件夹
系统 SHALL 支持将本地目录（包含子目录与文件）递归导入到当前保险箱路径，保留目录结构。

#### Scenario: 成功导入文件夹
- **WHEN** 用户在保险箱浏览器点击“导入明文文件夹”并选择一个本地目录
- **THEN** 系统在当前保险箱目录下创建同名根目录并递归导入其子内容
- **AND** 导入完成后刷新当前目录列表

### Requirement: 文件读取与预览/导出
系统 SHALL 支持对保险箱内文件进行读取，并向用户提供预览与导出能力。

#### Scenario: 文本预览
- **WHEN** 用户点击一个文件
- **THEN** 系统弹出操作面板，包含“预览（文本）”与“导出到临时目录”
- **WHEN** 用户选择“预览（文本）”
- **THEN** 系统读取文件的前 N 字节（例如 256KB 上限）并尝试以 UTF-8 展示；若无法解码则提示该文件不可文本预览

#### Scenario: 导出到临时目录
- **WHEN** 用户选择“导出到临时目录”
- **THEN** 系统将文件解密后的内容输出到应用临时目录，并提示导出结果（例如文件名/保存位置）

### Requirement: 进度反馈与取消
系统 SHALL 在导入/导出操作中提供最小可用的进度反馈与取消能力。

#### Scenario: 长任务可取消
- **WHEN** 导入/导出任务开始
- **THEN** 系统展示进度 UI（至少包括当前处理的文件名与已完成计数）
- **WHEN** 用户点击“取消”
- **THEN** 系统停止后续文件处理，并保持已完成部分不回滚

## MODIFIED Requirements
### Requirement: 保险箱文件浏览器交互
系统 SHALL 不再展示“(加密功能开发中)”或“文件读取功能开发中”的占位提示，而是执行真实导入/读取逻辑并处理错误提示。

## REMOVED Requirements
### Requirement: “开发中”占位提示
**Reason**: 功能需要投入实际使用，避免误导用户。
**Migration**: 无；直接替换为真实实现。
