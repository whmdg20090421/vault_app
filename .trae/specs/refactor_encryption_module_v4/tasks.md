# Tasks

- [x] Task 1: 实现文件导入与路径获取机制
  - [x] SubTask 1.1: 在「导入加密文件夹」或「导入加密文件」按钮点击时，唤起系统原生文件选择界面。
  - [x] SubTask 1.2: 用户选择文件/文件夹并确认后，将所选绝对路径提交至应用核心处理模块，触发独立加密任务的创建。

- [x] Task 2: 实现加密文件结构树定义与构建
  - [x] SubTask 2.1: 定义核心加密数据模型 `EncryptionNode`，包含 `folder` 和 `file` 两种类型。
  - [x] SubTask 2.2: 实现文件夹节点的字段：`taskId` (仅根节点), `name`, `type` (固定 folder), `isPaused`, `children`。
  - [x] SubTask 2.3: 实现文件节点的字段：`name`, `type` (固定 file), `size` (自动换算1~1000且保留1位小数, B/KB/MB/GB/TB), `rawSize` (原始字节), `status` (5种枚举值), `isPaused`。
  - [x] SubTask 2.4: 实现结构树构建流程：生成全局唯一 `taskId`，创建根节点并加入全局任务列表。
  - [x] SubTask 2.5: 递归遍历根节点下所有子文件与子文件夹，同步读取原始字节并换算 `size`。
  - [x] SubTask 2.6: 新建节点默认 `isPaused=false`，文件节点默认 `pending_waiting`。
  - [x] SubTask 2.7: 构建完成后，将全局任务列表采用JSON格式持久化至应用内部私有数据目录。
  - [x] SubTask 2.8: 新任务创建后，自动触发一次全局进度刷新，并启动动态刷新机制。

- [x] Task 3: 实现多线程加密执行与异常处理
  - [x] SubTask 3.1: 读取应用设置的加密线程数，启动对应数量的加密线程，并行处理所有 `status=pending_waiting` 且 `isPaused=false` 的文件节点。
  - [x] SubTask 3.2: 实现加密中断处理：应用退出/异常导致中断时，文件状态重置为 `pending_waiting`。重启后自动读取本地任务列表，对未完成未暂停的文件执行完整重新加密（不支持断点续传）。
  - [x] SubTask 3.3: 实现文件异常处理：检测到文件缺失/损坏/无法读取时，立即终止该文件加密，状态设为 `error`，在UI上标记异常并提示修复，且不影响其他正常文件加密。

- [x] Task 4: 实现加密状态更新与进度恢复
  - [x] SubTask 4.1: 单个文件开始加密时，将对应节点状态更新为 `encrypting`。
  - [x] SubTask 4.2: 单个文件加密完成后，将对应节点状态更新为 `completed`。
  - [x] SubTask 4.3: 应用重启时，读取本地全局任务列表，基于 `status` 和 `isPaused` 恢复进度。
  - [x] SubTask 4.4: 重启时自动扫描所有文件节点，将原文件系统中已不存在的文件状态自动更新为 `error`。

- [x] Task 5: 构建主页导航栏加密信息总览UI
  - [x] SubTask 5.1: 在主页顶部导航栏添加小型四色进度条图标，点击显示详细数值提示（已加密大小 / 总大小，自动换算单位）。
  - [x] SubTask 5.2: 实现数据计算逻辑：已加密总大小(绿)、正在加密总大小(黄)、待加密总大小(红)、已暂停/错误总大小(灰)，四者合计100%。
  - [x] SubTask 5.3: 实现进度条强制可视化规则：理论占比 <1% 时强制保持 1 像素最小长度，且不影响其他颜色正常比例。

- [x] Task 6: 构建层级化加密进度详情查看模态框
  - [x] SubTask 6.1: 添加「查看同步进度」UI按钮，点击后从下往上滑入弹出半屏模态框，展示所有正在进行的加密任务根节点（同级关系）。
  - [x] SubTask 6.2: 支持点击任意节点展开/折叠子目录，多级嵌套展开与实际文件系统一致。
  - [x] SubTask 6.3: 实现条目三行紧凑布局：第一行显示暂停/继续圆形按钮、名称（异常带红叹号）。
  - [x] SubTask 6.4: 第二行显示单条极细字节四色进度条（绿、黄、红、灰），严格遵循强制可视化规则（<1% 强制1像素）。
  - [x] SubTask 6.5: 第三行左侧显示当前条目整体加密百分比（保留1位小数，异常显示「存在异常」）；右侧显示加密数量统计（文件为1/1，文件夹为已加密/总文件数，不含文件夹本身）。
  - [x] SubTask 6.6: 实现条目级暂停/继续功能：点击暂停立即终止正在加密的线程，该节点及子节点设为 `isPaused=true`，`encrypting` 重置为 `pending_paused`。点击继续设为 `isPaused=false`，`pending_paused` 重新加入队列并设为 `pending_waiting`。父节点暂停覆盖子节点，子节点暂停不影响父/同级。
  - [x] SubTask 6.7: 实现长按删除功能：长按条目底部弹出操作菜单「移除加密」（异常条目多出「标记为已修复」）。点击后弹出二次确认。
  - [x] SubTask 6.8: 执行删除时，仅删除该节点及子节点，终止相关加密线程，从全局列表中永久移除，实时更新统计。
  - [x] SubTask 6.9: 执行「标记为已修复」时，将该节点及子节点的 `error` 状态重置为 `pending_waiting`，重新加入加密队列。

- [x] Task 7: 实现动态进度刷新机制
  - [x] SubTask 7.1: 定义前台显示状态与后台静默状态的监听逻辑。
  - [x] SubTask 7.2: 在后台静默状态下，每 5~10 秒随机一次刷新，仅更新内存数据并持久化，不重绘UI。
  - [x] SubTask 7.3: 在前台显示状态下，打开界面时立即全量刷新，随后每 0.5~1 秒随机一次高频刷新（更新内存和UI）。
  - [x] SubTask 7.4: 实现状态无缝切换：最后界面关闭回退到低频，任意界面打开切到高频。
  - [x] SubTask 7.5: 无活跃加密任务时（全部 `completed`），执行一次最终全量刷新，永久缓存本地，停止所有自动刷新任务。再次打开界面直接读取缓存。新任务创建才重新启动刷新。
  - [x] SubTask 7.6: 独立监听线程仅负责数据计算和更新，不直接操作 UI，确保任何情况持久化正常。

- [x] Task 8: 实现任务完成与自动归档
  - [x] SubTask 8.1: 当加密任务根节点下所有文件节点均为 `completed` 时，判定任务完成。
  - [x] SubTask 8.2: 任务完成后执行目录存在性检查：遍历根节点下所有文件/文件夹，验证实际是否存在。
  - [x] SubTask 8.3: 若全存在，将任务移至加密历史记录中永久保存。若缺失，保留在进行中列表，缺失文件设为 `error`，UI标记异常提示手动处理。
  - [x] SubTask 8.4: 实现历史记录列表展示，条目布局一致（隐藏暂停和进度条，显示完成时间）。
  - [x] SubTask 8.5: 历史记录支持长按弹出「删除历史」，二次确认后永久移除进度数据（不删文件和索引）。

- [x] Task 9: 实现加密完成后处理（哈希入库）
  - [x] SubTask 9.1: 单个文件加密完成后，计算加密后文件的哈希值。
  - [x] SubTask 9.2: 将哈希值与文件唯一标识（`taskId + 相对路径`）关联写入本地哈希索引库。
