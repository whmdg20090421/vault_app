- [ ] 核心加密数据模型 (`EncryptionNode`) 支持文件夹和文件的层级序列化，且包含完整的状态机枚举 (`pending_waiting`, `pending_paused`, `encrypting`, `completed`, `error`)。
- [ ] 导入文件夹时，系统为其分配全局唯一 `taskId`，正确递归生成包含 `size` (自动换算) 和 `rawSize` 的结构树。
- [ ] 多线程加密执行器按设置数量并行启动，仅处理 `status=pending_waiting` 且 `isPaused=false` 的文件节点。
- [ ] 单个文件加密中断（如强退），下次启动时重置为 `pending_waiting`。文件意外缺失或损坏则更新为 `error` 且不影响其他文件。
- [ ] 主页顶部导航栏出现四色进度条图标，展示待加密、正在加密、已完成和暂停/错误的 `rawSize` 占比，且占比 <1% 的颜色段强制显示 1 像素。
- [ ] 提供「查看同步进度」按钮，点击滑出半屏模态框，展示所有正在进行的加密任务，且结构树支持同级显示与嵌套展开/折叠。
- [ ] 模态框内的条目（三行布局）正确显示圆形暂停/继续按钮、名称（异常时带红叹号）、极细进度条及整体进度百分比和完成数统计（文件为 1/1，文件夹为已完成文件数/总文件数）。
- [ ] 模态框内的条目级暂停/继续按钮生效，正确级联至所有子节点，且父节点暂停覆盖子节点状态。
- [ ] 长按任意条目可唤出底部菜单「移除加密」（异常时含「标记为已修复」），删除操作仅移除该节点及其子节点，不影响父级与同级。
- [ ] 动态进度刷新机制工作正常：打开进度界面时高频刷新（0.5-1s），关闭界面后自动切回低频静默（5-10s），所有任务完成后停止监听。
- [ ] 任务完成后，系统自动验证原文件系统存在性，存在则移入加密历史记录。
- [ ] 加密完成后，将加密文件的哈希值与 `taskId + 相对路径` 关联写入本地哈希索引库。
