# Fix Cloud Sync State and UI Spec

## Why
目前在应用中，首页刷新的云端文件数量和差异文件数量统计不准确或始终为0。在云端文件浏览中，即使文件已同步，仍然显示“未同步”。点击开始同步后按钮会一直转圈直至重启应用，且同步进度缺乏速度和剩余时间的显示，导致用户体验不佳。

## What Changes
- 修改首页的刷新逻辑，精确统计云端（加密）文件夹内的文件数量，并计算本地与云端真实存在差异的文件数量。
- 修复云端文件浏览器中“未同步”状态的判断逻辑，确保能够正确识别云端文件和本地文件的一致性。
- 修复点击“开始同步”后无限转圈的问题。调整为：点击后进行差异计算（统计需要上传和下载的文件），计算完成后恢复按钮状态，关闭弹窗，并将任务推送到同步任务进度中。
- 在同步任务进度管理器中增加对传输速度（上传/下载）和预计剩余时间的计算和显示。

## Impact
- Affected specs: 首页统计显示、云端文件浏览器状态显示、同步设置对话框、同步进度详情显示。
- Affected code: `lib/cloud_drive/` 及首页相关逻辑（可能在 `lib/pages/home_page.dart` 或类似文件中）、`SyncSettingsDialog`、`CloudDriveProgressManager`。

## MODIFIED Requirements
### Requirement: 首页统计刷新
- **WHEN** 用户点击刷新按钮
- **THEN** 系统应通过网络请求获取云端同步目录的实际文件数量，并与本地索引/文件进行对比，得出准确的差异文件数量。

### Requirement: 云端文件状态显示
- **WHEN** 用户在云端文件浏览器中查看文件列表
- **THEN** 系统应使用与同步引擎一致的差异判断逻辑（如对比文件大小），正确显示文件的“已同步”或“未同步”状态。

### Requirement: 开始同步交互
- **WHEN** 用户在同步设置中点击“开始同步”
- **THEN** 系统开始计算差异文件，界面显示加载状态；计算完毕并将任务添加到进度队列后，加载状态结束，弹窗关闭。

### Requirement: 同步进度显示
- **WHEN** 用户查看同步进度详情
- **THEN** 界面上应显示当前任务的传输速度（如 KB/s 或 MB/s）和预计剩余时间。
