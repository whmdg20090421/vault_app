# 优化 UI、主题、网络连接与自定义背景 Spec

## 为什么 (Why)
当前项目存在几个问题：
1. 主题样式需要重新调整并增加纯黑主题，同时用户希望在所有主题下都能支持完全自定义的图片背景；
2. 加密空间的系统返回键逻辑不合理，导致直接退出整个界面；
3. 主页“数据概览”UI 布局和数据展示不符合预期；
4. 文件列表大小展示不直观；
5. WebDAV 云盘连接存在证书校验问题；
6. 核心问题：应用中存在部分硬编码数据或占位符，不符合“真实环境读取”和“数据真实化”的要求。

## 变更内容 (What Changes)
- **主题与自定义背景重构**：
  - 修改赛博朋克主题为圆角风格，背景改为彩色基调渐变。新增“纯黑”极简主题。
  - 在“关于”页面（或设置页）中新增“背景自定义”入口，包含开关、选择图片、图片透明度滑动条和 UI 透明度滑动条。
  - 选中的图片将持久化缓存到本地数据目录。开启开关后，无视当前主题，强制将应用全局背景替换为该缓存图片，并应用对应的透明度参数。
- **导航修复**：使用 `PopScope` 拦截 Android 返回键，实现加密空间逐级目录返回，仅在根目录允许退出。
- **数据概览重构**：主页图表改为横向满铺面板，使用线性进度条展示已加密 vs 未加密数据。
- **数据真实化与缓存**：**[核心]** 检查程序中所有计算数据的模块（如统计、列表、进度条），确保全部从真实环境读取（如扫描本地文件、读取状态管理器），并实现缓存机制。对于空数据，必须显示“暂无数据”空态，严禁使用硬编码占位符。
- **单位换算**：实现 `formatBytes` 并在文件列表等处应用，自动换算 KB/MB/GB。
- **网络修复与 WebDAV 核心逻辑实现**：
  - WebDAV 客户端配置忽略 SSL 证书校验，并确保 Android 原生配置支持明文传输和网络权限。
  - 参考 `dom111/webdav-js` 的核心逻辑（如构造 `PROPFIND`、`DELETE`、`MKCOL`），使用 Dart 原生网络库（Dio/HttpClient）和 XML 解析库等效复刻 WebDAV 协议逻辑。
  - 实现云盘的真实浏览（解析 XML 并映射为 Flutter 模型）、真实删除功能，并建立基础的云端与本地状态同步机制（对比并标记文件的“已同步/未同步”状态）。
- **全局自检与自动化发布**：开发完成后，全局检查代码、逻辑与 UI。随后提交代码、触发 CI 编译、自动修复报错，最终发布 `1.2.2` 正式版（Release 描述仅包含当前版本更新）。

## 影响范围 (Impact)
- 影响模块：主题管理，全局背景渲染，文件系统导航，主页概览统计，WebDAV 客户端，全局数据展示组件。
- 影响代码：
  - `lib/theme/` 及全局包裹背景的 Widget
  - `lib/about_page.dart` 或设置页面（新增自定义背景入口）
  - `lib/encryption/vault_explorer_page.dart`
  - `lib/home_page.dart`
  - `lib/cloud_drive/webdav_client_service.dart`
  - `lib/services/stats_service.dart` 

## 新增需求 (ADDED Requirements)
### 需求：全局自定义图片背景
系统必须支持用户从本地选择图片作为全局应用背景，并提供 UI 和图片的透明度调节。

#### 场景：成功案例
- **当** 用户在“背景自定义”中开启开关，选择一张图片，并设置 UI 透明度为 0.8 时
- **则** 无论当前处于“纯黑”还是“赛博朋克”主题，应用的根级背景都会被替换为该图片，且应用内的卡片/面板呈现 80% 透明度以透出底层图片。

## 修改的需求 (MODIFIED Requirements)
### 需求：主页数据概览重构 (结合真实性)
替换现有的饼图，使用横向满铺的线性进度条，展示从 `StatsService` 读取并缓存的真实数据。
