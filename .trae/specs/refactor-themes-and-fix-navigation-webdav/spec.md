# 优化 UI、主题与网络连接 Spec

## 为什么 (Why)
当前项目存在几个问题：主题样式（赛博朋克主题的视觉效果）需要重新调整并增加纯黑主题；加密空间的系统返回键（侧滑/物理按键）逻辑不合理，导致直接退出整个界面而不是返回上一级目录；主页的“数据概览”UI布局和数据展示不符合预期；文件列表大小展示不够直观；WebDAV云盘连接时存在潜在的SSL证书校验拦截问题，导致DNS解析失败。我们需要修复这些问题以提升用户体验和软件稳定性。

## 变更内容 (What Changes)
- 重构现有赛博朋克主题：修改直角UI为圆角，背景调整为具有科技感的渐变或彩色基调。
- 新增“黑色”主题：实现一套纯黑背景主题，体现高级极简质感，严禁沿用赛博朋克的发光边框。
- 修复侧滑/物理返回键逻辑：在多级文件夹浏览中，使用 `PopScope` 拦截返回事件，仅在根目录时允许退出，其余情况返回上一级。
- 重构主页“数据概览”UI：将图表替换为横向满铺面板，包含文本和条形进度条（绿红双色），并接入真实的统计逻辑（已加密文件 vs 未加密文件）。
- 优化文件列表大小展示：增加 `formatBytes` 函数，自动将字节数换算为 KB、MB、GB 等单位并保留两位小数。
- 修复 WebDAV 连接问题：在 `Dio` 或 `HttpClient` 初始化中添加忽略 SSL 证书校验的配置，并确保 `AndroidManifest.xml` 中存在 `android:usesCleartextTraffic="true"` 及相关权限，以解决网络拦截与DNS解析问题。

## 影响范围 (Impact)
- Affected specs: 主题切换，文件浏览导航，主页概览面板，WebDAV 连接。
- Affected code: 
  - `lib/theme/` (或相关的主题配置)
  - `lib/encryption/vault_explorer_page.dart` (侧滑返回及文件大小格式化)
  - `lib/home_page.dart` (数据概览重构)
  - `lib/cloud_drive/webdav_client_service.dart` (忽略证书校验)
  - `android/app/src/main/AndroidManifest.xml` (网络配置)

## 新增需求 (ADDED Requirements)
### 需求：纯黑主题与自适应返回拦截
系统必须提供一个新的纯黑极简主题。在浏览加密文件系统时，物理返回操作必须能够根据当前的路径深度决定是返回上一级还是退出界面。

#### 场景：成功案例
- **当** 用户处于加密空间的 `/folder1/folder2` 目录时，触发侧滑返回
- **则** 界面返回到 `/folder1` 而不是直接退出。

## 修改的需求 (MODIFIED Requirements)
### 需求：主页数据概览重构
替换现有的饼图，使用横向满铺的线性进度条，并直接展示真实的 `StatsService` 数据大小与百分比。

### 需求：WebDAV 证书忽略
必须在网络请求客户端层面忽略所有的 SSL 证书错误，以规避某些被系统拦截或由于自签名证书导致的假性 DNS 解析失败。
