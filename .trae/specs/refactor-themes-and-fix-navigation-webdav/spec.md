# 优化 UI、主题、网络连接与数据真实化 Spec

## 为什么 (Why)
当前项目存在几个问题：
1. 主题样式（赛博朋克主题的视觉效果）需要重新调整并增加纯黑主题；
2. 加密空间的系统返回键逻辑不合理，导致直接退出整个界面；
3. 主页“数据概览”UI 布局和数据展示不符合预期；
4. 文件列表大小展示不直观；
5. WebDAV 云盘连接存在证书校验问题；
6. 核心问题：应用中存在部分硬编码数据或占位符，不符合“真实环境读取”和“数据真实化”的要求。

## 变更内容 (What Changes)
- **主题重构**：修改赛博朋克主题为圆角风格，背景改为彩色基调渐变。新增“纯黑”极简主题，禁用发光边框。
- **导航修复**：使用 `PopScope` 拦截 Android 返回键，实现加密空间逐级目录返回，仅在根目录允许退出。
- **数据概览重构**：主页图表改为横向满铺面板，使用线性进度条展示已加密 vs 未加密数据。
- **数据真实化与缓存**：**[核心]** 检查程序中所有计算数据的模块（如统计、列表、进度条），确保全部从真实环境读取（如扫描本地文件、读取状态管理器），并实现缓存机制。对于空数据，必须显示“暂无数据”空态，严禁使用硬编码占位符。
- **单位换算**：实现 `formatBytes` 并在文件列表等处应用，自动换算 KB/MB/GB。
- **网络修复**：WebDAV 客户端配置忽略 SSL 证书校验，并确保 Android 原生配置支持明文传输和网络权限。

## 影响范围 (Impact)
- 影响模块：主题管理，文件系统导航，主页概览统计，WebDAV 客户端，全局数据展示组件。
- 影响代码：
  - `lib/theme/`
  - `lib/encryption/vault_explorer_page.dart`
  - `lib/home_page.dart`
  - `lib/cloud_drive/webdav_client_service.dart`
  - `lib/services/stats_service.dart` (数据统计核心)
  - `android/app/src/main/AndroidManifest.xml`

## 新增需求 (ADDED Requirements)
### 需求：全量数据真实化与空态处理
系统内所有涉及数据展示的模块（图表、列表、统计数值等）必须接入真实数据源并具备缓存能力。

#### 场景：成功案例
- **当** 用户打开主页且没有任何加密文件时
- **则** 数据概览面板显示“暂无数据”或全 0 真实统计，而非预设的模拟数值。

## 修改的需求 (MODIFIED Requirements)
### 需求：主页数据概览重构 (结合真实性)
替换现有的饼图，使用横向满铺的线性进度条，展示从 `StatsService` 读取并缓存的真实数据。

### 需求：WebDAV 证书忽略
在网络层忽略 SSL 证书校验，确保底层连接畅通。
