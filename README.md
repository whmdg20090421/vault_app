# 天眼·艨艟战舰 (Vault App)

这是一个端到端加密云盘与本地保险箱的 Flutter 应用。结合硬件级密钥存储与强一致性加密算法，提供最高规格的数据隐私保护。

## 更新历史 (Changelog)

### 1.1.3 (2026-04-16)

#### ✨ Features & Security
- **底层加密核心架构重构**：深入参考了 DroidFS (gocryptfs) 的底层逻辑，用 Dart 原生完全重构了虚拟文件系统 (VFS) 的加密引擎。
- **目录结构 1:1 映射**：实现了明文目录与密文目录的 1:1 结构映射，去除了此前扁平化存储带来的弊端，大幅优化了目录层级的管理与未来网盘增量同步的效率。
- **确定性文件名加密机制**：引入了基于固定 IV 的 AES-256-GCM 算法对文件名进行加密，并进行 Base64Url 编码。确保同一个文件名在同一路径下加密后的密文名始终一致，彻底解决同步时的文件名冲突与重复识别问题。
- **流式分块加密 (Chunked Streaming)**：数据加密改用分块处理（默认 64KB 为一块）。每个加密文件头部随机生成并写入 16 字节的唯一 File ID。后续每个数据块的 Nonce 均由 `(File ID + 块序号)` 动态派生生成，这不仅杜绝了重放攻击，且大大减少了 Nonce 的存储开销。
- **多线程/Isolate 性能提升**：为了防止在进行文件加解密及高强度密钥派生（Argon2id/Scrypt/PBKDF2）时阻塞主线程导致 UI 卡顿，将密集的 CPU 运算任务全部通过 `Isolate.run()` 移交至后台线程并发执行。

#### 🚀 Performance & UI
- **极致的保险箱解锁速度**：彻底解决了此前空保险箱输入密码后解锁耗时超过 1 分钟的性能问题。通过下调默认的 KDF 派生参数（如 Scrypt 的 N 从 2^20 降至 16384，Argon2 内存降至 4096）并配合 Isolate 异步计算，目前解锁耗时已被控制在 1-3 秒之内，实现秒级开启。
- **主页数据概览真实化重构**：废弃了主页的硬编码假数据。引入了全新的 `StatsService` 统计服务。该服务会自动扫描本地保险箱与未加密缓存区，计算真实存储体积。
- **缓存策略与单位换算**：统计结果被持久化缓存至 `SharedPreferences`。应用启动时可实现“零延迟”读取渲染，仅在发生实际的加解密操作、云端同步或用户手动下拉刷新时才在后台重新统计。图表数据现在能智能换算 MB 与 GB 显示。
- **UI 布局更新**：主页改版为左右横向结构（Row），左侧放置等宽高的正方形数据统计饼图卡片，右侧预留未来功能的展示空间。
- **新建文件夹功能**：在保险箱内部的虚拟文件浏览器中，正式加入了“新建文件夹”按钮，补全了基础的密文空间目录管理能力。
- **日志文档全面升级**：为了项目的长期可维护性，应用内及代码仓库的 `README.md` 与 `CHANGELOG.md` 记录规范进行了大幅完善，历史版本更新细节变得前所未有的详尽。

### 1.1.0 (2026-04-16)

#### ✨ Core Vault Features
- **独立加密保险箱管理**：重磅推出了核心的本地保险箱（Vault）功能：支持从零创建、参数配置、列表总览以及验证解锁进入保险箱等一系列完整闭环操作。
- **高自由度加密协议矩阵**：给予用户最大的加密控制权，支持自由选择顶尖的对称加密算法（AES-256-GCM、ChaCha20-Poly1305），并搭配业界公认最强的抗硬件穷举 KDF（Key Derivation Function）方案（PBKDF2、Argon2id、Scrypt）。所有算法参数（如内存消耗、迭代次数、并行度）均开放给极客用户自行调节。
- **明文配置文件机制**：在每个保险箱的根目录下生成明文的 `vault_config.json` 文件。该文件独立记录了该保险箱所使用的算法、KDF、具体参数、是否开启文件名加密，以及用于验证密码正确性的校验密文块，使得保险箱具备跨设备迁移与恢复的自描述能力。
- **硬件加密性能基准测试 (Benchmark)**：在配置页面右上角新增了一键式硬件性能评估工具。它会在本地生成 500MB 的随机测试数据，模拟最高强度的加密写入，最终为用户呈现该设备的真实平均加密速度（MB/s），辅助用户在安全与速度之间寻找完美平衡点。
- **赛博朋克主题全面升级**：废弃了早期简陋的 UI，将全局的“赛博朋克”主题升级为极具冲击力的高对比度霓虹配色，并对全部加密弹窗与设置面板进行了深度深度适配与微调。

### 1.0.1 (2026-04-16)

#### ✨ WebDAV & Security Infrastructure
- **WebDAV 全生命周期管理**：在“云盘”功能区引入了完整的 WebDAV 节点配置管理能力，支持增、删、改、查等全套操作，为后续的网盘透明加解密打下基石。
- **硬件级安全等级检测**：新增应用自检功能，根据 Android 设备是否支持硬件支持的 Keystore 密钥库，自动将其划分为安全等级 Level 1 或 Level 2，并在 UI 侧向用户提供直观的图形化警告与科普提示。
- **混合持久化存储方案**：配置了本地数据的混合存储能力——非敏感的基础设置字段存储至普通的本地 JSON；而高度敏感的 WebDAV 密码凭证则强制通过 `flutter_secure_storage` 保存于系统底层的安全加密容器中。

#### 🐛 Bug Fixes & CI
- **主题状态同步修复**：修复了在底部导航栏进行频繁切换时，“设置”页面内的“主题切换”按钮状态未实时跟随应用当前主题刷新的 UI 状态不同步问题。
- **CI 签名配置劫持修复**：彻底解决了 Android 生产构建打包时，GitHub Actions 环境无法正确读取仓库 Secrets（如 `KEY_STORE_BASE64`, `KEY_ALIAS`, `KEY_PASSWORD`, `STORE_PASSWORD`）并执行 `assembleRelease` 生产签名的问题。现在可确保每次云端编译出的 APK 签名哈希绝对一致，以支持用户端的无缝覆盖安装升级。
- **工作流安全锁**：修改了 GitHub Actions 打包工作流逻辑，当缺失签名密钥时将强制拦截并终止构建流程，避免发布无签名的无效包。
